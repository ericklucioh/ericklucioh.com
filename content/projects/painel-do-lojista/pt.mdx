## O problema que ele resolve

O sistema centraliza operacoes comuns de uma loja em um unico painel:

- login com controle de sessao;
- gestao de usuarios por perfil;
- cadastro e manutencao de produtos;
- movimentacao de estoque;
- abertura de caixa;
- registro de vendas;
- emissao de resumo/recibo da venda.

Em vez de tratar cada tela como uma demo visual, a implementacao conecta
frontend, backend e banco de dados em um fluxo unico de operacao.

## Tecnologias usadas

### Frontend

- **Next.js + React + TypeScript** para a aplicacao web.
- **App Router** para separar fluxo de autenticacao e dashboard.
- **React Hook Form + Zod** para formularios com validacao previsivel.
- **Tailwind CSS + shadcn/ui** para acelerar a construcao da interface.

### Backend

- **Node.js + Express + TypeScript** para a API.
- **Prisma** para modelagem, migrations e acesso ao banco.
- **MySQL** como base relacional para usuarios, produtos, estoque e vendas.
- **JWT + refresh token** para autenticacao e renovacao de sessao.

### Infra e qualidade

- **Docker Compose** para subir frontend, backend, MySQL e Adminer.
- **ESLint, Prettier e testes** para manter consistencia e validar o fluxo.
- **GitHub Actions** para automacao de checks no repositorio.

## O que eu aprendi

Esse projeto foi importante porque me forcou a ir alem do CRUD basico.

- **Separacao de responsabilidades**: frontend, backend e regras de negocio precisam de fronteiras claras para o projeto continuar escalavel.
- **Autenticacao de verdade**: nao basta "fazer login"; e preciso pensar em cookies, refresh token, expiracao, middleware e autorizacao por papel.
- **Modelagem de fluxo**: estoque, caixa e venda se conectam. Se a modelagem estiver errada, a aplicacao inteira fica inconsistente.
- **Ambiente reproduzivel**: documentacao, `.env.example`, seed e Docker fazem muita diferenca para quem vai rodar o projeto pela primeira vez.
- **Desafio tecnico bom se ganha na execucao**: nao so pela stack, mas pela capacidade de transformar requisitos espalhados em um sistema coerente.
