## O problema que ele resolve

O sistema centraliza operações comuns de uma loja em um único painel:

- login com controle de sessão;
- gestão de usuários por perfil;
- cadastro e manutenção de produtos;
- movimentação de estoque;
- abertura de caixa;
- registro de vendas;
- emissão de resumo/recibo da venda.

Em vez de tratar cada tela como uma demo visual, a implementação conecta
frontend, backend e banco de dados em um fluxo único de operação.

## Tecnologias usadas

### Frontend

- **Next.js + React + TypeScript** para a aplicação web.
- **App Router** para separar fluxo de autenticação e dashboard.
- **React Hook Form + Zod** para formulários com validação previsível.
- **Tailwind CSS + shadcn/ui** para acelerar a construção da interface.

### Backend

- **Node.js + Express + TypeScript** para a API.
- **Prisma** para modelagem, migrations e acesso ao banco.
- **MySQL** como base relacional para usuários, produtos, estoque e vendas.
- **JWT + refresh token** para autenticação e renovação de sessão.

### Infra e qualidade

- **Docker Compose** para subir frontend, backend, MySQL e Adminer.
- **ESLint, Prettier e testes** para manter consistência e validar o fluxo.
- **GitHub Actions** para automação de checks no repositório.

## O que eu aprendi

Esse projeto foi importante porque me forçou a ir além do CRUD básico.

- **Separação de responsabilidades**: frontend, backend e regras de negócio precisam de fronteiras claras para o projeto continuar escalável.
- **Autenticação de verdade**: não basta "fazer login"; é preciso pensar em cookies, refresh token, expiração, middleware e autorização por papel.
- **Modelagem de fluxo**: estoque, caixa e venda se conectam. Se a modelagem estiver errada, a aplicação inteira fica inconsistente.
- **Ambiente reproduzível**: documentação, `.env.example`, seed e Docker fazem muita diferença para quem vai rodar o projeto pela primeira vez.
- **Desafio técnico bom se ganha na execução**: não só pela stack, mas pela capacidade de transformar requisitos espalhados em um sistema coerente.
